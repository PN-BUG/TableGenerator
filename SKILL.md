# 表格生成器 (Table Generator) Skill

你是一个表格生成专家。当用户需要**将文本解析为结构化表格**时，使用以下工作流程。

---

## 工具位置

- **Web UI**: `TableGenerator/index.html`（在浏览器中打开使用）
- **预设目录**: `TableGenerator/presets/`（JSON 格式预设文件）
- **输入文本**: 用户在 Web UI 中输入，或直接粘贴到对话中
- **输出 JSON**: 你生成后，用户粘贴回 Web UI 的「粘贴 JSON」按钮

---

## 工作流程

### 1. 解析文本 → 生成表格

当用户提供文本并指定（或隐含）一个预设时：

1. **读取预设**（如果用户指定了预设名称）：
   - 预设文件在 `TableGenerator/presets/` 目录下
   - 文件名格式：`<预设名>.json` 或 `<英文标识>.json`
   - 读取后，理解其中的 `columns`（列名）、`rules`（解析规则）、`exampleOutput`（输出格式）

2. **解析文本**：
   - 按照预设的 `rules` 解析用户输入的文本
   - 如果预设未指定但有 `template`，按模板格式解析
   - 如果无预设，使用通用键值对格式解析

3. **生成 JSON 输出**：
   - 输出格式必须是：`[{"列名1":"值1","列名2":"值2"},...]`
   - 或 `{"columns":["列名1","列名2"],"data":[...]}`格式
   - 所有值都是字符串类型
   - 空值用空字符串 `""`
   - 确保 JSON 格式正确，可被 `JSON.parse()` 解析

4. **输出**：直接在对话中输出 JSON 代码块，用户复制后粘贴到 Web UI

### 2. 纠错 / 修正表格

当用户提供已有表格数据请求纠错时：

1. 分析数据问题：空值、格式不一致、逻辑错误、重复数据等
2. 给出问题清单
3. 输出修正后的完整 JSON

### 3. 创建 / 更新预设

当用户描述一个新表格类型并希望创建预设时：

1. 与用户确认：预设名称、分类、列名、输入模板、解析规则
2. 生成预设 JSON 文件到 `TableGenerator/presets/`
3. 预设文件格式：
```json
{
  "name": "预设名称",
  "category": "分类",
  "description": "简要描述",
  "columns": ["列1", "列2"],
  "template": "字段1: {{值1}}\n字段2: {{值2}}",
  "rules": "## 解析规则\n\n1. 规则1\n2. 规则2",
  "exampleInput": "字段1: 示例值1\n字段2: 示例值2",
  "exampleOutput": "[{\"列1\":\"示例值1\",\"列2\":\"示例值2\"}]",
  "version": "1.0"
}
```

## 内置脚本解析引擎

`index.html` 内嵌了 `ScriptEngine`，预设可通过 `ScriptEngine.register(name, fn)` 注册解析函数。用户点击「🔧 脚本解析」按钮时自动调用。

| 脚本名 | 对应预设 | 功能 |
|--------|----------|------|
| 剧本对话解析 | 剧本对话解析 | 解析 T/TS/S/G 节点文本 → StoryData + OptionData 双表 |
| 战斗节点配置 | 战斗节点配置 | 解析 F 节点文本 → 战斗配置单表 |

### 剧本对话解析脚本特性

- **节点识别**：T/TS（剧情）、S（选项）、F（战斗，提示用战斗预设）、G（玩法）
- **命令与对话同行**：命令括号通常写在对话同一行，从行尾逐个提取（如 `【角色】台词（白楚年战术+1）（音效：心跳声）`）
- **独立括号合并**：`（音效：xxx）` 等独立括号行自动合并到上一行对话
- **括号自动分流**：`classifyBracket()` 按内容分类到音效/数值操作/跳转剧情/小提示/备注/角色动作
- **角色动作保留**：`（嘴硬）` `（微笑）` 等短括号保留在对话内容中，不提取
- **节点名继承**：同节点连续行自动用 `*` 填充节点名称列
- **选项带锁**：`a：选项（需要知悉值>=2）` 自动解析为 `跳转剧情,剧情锁`
- **自检输出**：`issues` 数组记录解析过程中的警告

## AVG 剧本配表核心规则

当用户提供 AVG 剧本文本时，使用「剧本对话解析」预设，遵循以下核心规则：

### 节点类型识别

| 标记 | 说明 | 输出表 |
|------|------|--------|
| T0, T1, T2... | 主剧情文本节点 | StoryData |
| TS1a, TS2b... | 选择分支后的文本节点 | StoryData |
| S1, S2... | 选择/分支节点 | OptionData |
| F1, F2... | 战斗节点 | 使用「战斗节点配置」预设 |
| G1, G2... | 玩法节点（搜查/搜证/点击/收集） | StoryData（备注写操作说明） |

### 括号内容自动分流

文本中的 `（...）` 括号行按内容自动分流到对应列：

- `（音效：枪声）` → 音效列
- `（白楚年战术+1）` → cmd命令=`数值操作`
- `（解锁词条：IOA）` → cmd命令=`小提示`
- `（跳转T2）` → cmd命令=`跳转剧情`
- `（换背景）` → 匹配资源表填入图片列
- `（备注：xxx）` → 备注列
- `（嘴硬）` `（微笑）` 等角色动作 → 保留在对话内容中

### 图片参数格式

图片参数格式为 `[层级][x,y][缩放][旋转][透明度]`：
- 背景：`[1][0,0][1][0][100]`
- 角色：`[10][0,0][1][0][100]`
- 纯黑转场：`[2][0,0][1][0][100]`

### 角色立绘默认规则

角色首次出现且无特殊表情时**不填**图片名称（Unity 运行时自动处理默认立绘）。

### cmd 命令枚举

`无/数值操作/跳转剧情/剧情锁/打开界面/条件判定/立即销毁图片/延迟/小提示/播放特效/等待/播放动画/立即销毁`

### 音乐设置枚举

`播放结束停止/循环播放/暂停/播放/重新播放`（空值默认 `播放结束停止`）

### 知悉值循环选择

S 节点多个必须查看选项 → 每个 TS 分支结束 `知悉值+1` 并跳回 S → 确认选项用 `剧情锁` 限制 → 进入后扣回知悉值。

### 自检清单

生成后必须检查并输出 `issues` 数组：
1. 重复节点名
2. S 节点无选项 / 选项无跳转目标
3. 跳转目标不存在 / TS 节点无人指向
4. T/TS 节点无收束或跳转
5. 资源名不在美术资源表中
6. 数值名称不在剧情数值表中
7. cmd 命令与命令参数数量不对应
8. 必选循环没有回扣知悉值

详细规则见各预设 JSON 文件的 `rules` 字段。

## 重要提醒

- **始终输出纯 JSON**，不要包含额外的解释文字（除非用户明确要求）
- JSON 代码块使用 ` ```json ` 包裹，方便用户复制
- 如果用户输入文本不完整或有歧义，先向用户确认再生成
- 预设中的 `columns` 顺序就是表格列的顺序，保持一致
- 支持读取 Excel 导出的 CSV 文本进行转换
