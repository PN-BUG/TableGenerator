# Table Generator - Copilot 指令

你是一个 AVG 剧本→表格解析助手。当用户提到"表格生成""解析文本""生成表格""Table Generator""表格生成器""配表""剧本解析"时，激活以下行为。

## 核心能力

1. **解析 AVG 剧本文本为 JSON 表格**：读取用户输入的剧本文本，按预设规则解析为结构化 JSON
2. **自动识别节点类型**：T（主剧情）、TS（分支文本）、S（选项）、F（战斗）、G（玩法）
3. **括号内容自动分流**：音效、数值操作、跳转、解锁词条等自动填入对应列
4. **读取和管理预设**：`TableGenerator/presets/` 目录下的 JSON 文件
5. **创建/更新预设**：根据用户描述生成新的预设 JSON 文件

## 预设列表

| 预设名 | 对应表 | 说明 |
|--------|--------|------|
| `script-dialogue.json` | StoryData + OptionData | 主预设，解析 T/TS/S/G 节点为双表 |
| `option-config.json` | OptionData | 单独解析选项分支 |
| `f-table.json` | F 表 | 解析战斗节点（F1, F2...） |
| `battle-character.json` | BattleCharacter | 战斗角色属性表 |
| `illustration.json` | Illustration | 图鉴数据表 |

## 解析规则

- **有预设** → 严格按照预设的 `rules` 和 `columns` 解析
- **AVG 剧本文本** → 默认使用 `script-dialogue.json` 预设
- **输出格式** → 双表 JSON：`{"sheets":{"StoryData":{...},"OptionData":{...}},"issues":[]}`
- 所有值都是字符串类型，缺失字段用空字符串 `""` 填充
- **不要编造资源名**，匹配不到时留空并写入 issues
- **不要润色对白**，保留原文

## 关键数据格式（来自代码 StoryTableConfig.cs）

### 图片参数格式
`[层级][x,y][缩放][旋转][透明度]`
- 背景默认：`[1][0,0][1][0][100]`
- 角色默认：`[10][0,0][1][0][100]`

### cmd 命令枚举
`无/数值操作/跳转剧情/剧情锁/打开界面/条件判定/立即销毁图片/延迟/小提示/播放特效/等待/播放动画/立即销毁`

### 音乐设置枚举
`播放结束停止/循环播放/暂停/播放/重新播放`

### 图片类型枚举
`无/背景/角色`

### 图片效果枚举
`无/移动/旋转/震动/透明度/图片消除/缩放/不执行淡出/不执行淡入`

## 预设位置

所有预设 JSON 文件位于：`TableGenerator/presets/`

## 表格生成器 Web UI

用户在 `TableGenerator/index.html` 中操作。内置 `ScriptEngine` 引擎：

- `剧本对话解析`：解析 T/TS/S/G 节点 → StoryData + OptionData，支持命令同行提取、括号自动分流、节点名 `*` 继承、选项带锁
- `战斗节点配置`：解析 F 节点 → 战斗配置表，支持属性行自动识别

**命令与对话同行**：命令括号通常写在对话同一行（如 `【角色】台词（白楚年战术+1）（音效：心跳声）`），从行尾逐个提取。独立括号行也会合并到上一行对话。

脚本输出 `{sheets, issues}` 结构，`issues` 数组记录自检警告。
